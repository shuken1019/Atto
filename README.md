# Atto

Atto는 패션 커머스 운영 전반을 직접 구현한 프로젝트입니다. 사용자 영역에서는 상품 탐색, 상세 조회, 장바구니, 주문, 마이페이지 흐름을 제공하고, 관리자 영역에서는 상품 등록, 주문 관리, 배너 관리, 회원 관리까지 다룰 수 있도록 구성했습니다. 프론트엔드와 백엔드를 분리해 실제 서비스에 가까운 구조로 설계했으며, 커머스 서비스의 핵심 기능과 운영 관점을 함께 담는 데 집중했습니다.

## 프로젝트 개요

- 사용자 쇼핑 경험과 관리자 운영 기능을 하나의 서비스 흐름으로 연결한 패션 커머스 프로젝트
- React + TypeScript 기반 프론트엔드와 Express + MySQL 기반 백엔드를 분리해 구현
- 상품 조회부터 주문 생성, 회원 인증, 관리자 상품/주문 운영까지 커머스 서비스의 주요 기능을 직접 설계 및 구현

## 주요 기능

### 사용자 기능

- 메인 배너, 라이브 상품, 베스트셀러, 신상품 중심의 홈 화면 구성
- 상품 목록 조회, 상세 페이지 조회, 옵션 선택, 장바구니 담기
- 로그인, 회원가입, 계정 찾기
- 찜 기능과 마이페이지 기반 주문/배송지 관리
- 무통장입금 기준의 결제 생성 및 주문 생성 흐름 구현
- 주문 내역 조회와 주문 취소, 교환/환불 요청 화면 구성

### 관리자 기능

- 관리자 전용 라우팅과 권한 분기 처리
- 상품 등록, 수정, 공개 여부 관리, 삭제 기능
- 주문 목록 조회, 결제 완료 처리, 주문 상태 변경
- 사용자 목록 조회 및 권한 변경
- 메인 배너 문구/이미지 관리
- 대시보드 기반 주문 현황 및 재고 현황 확인

## 기술 스택

### Frontend

- React 19
- TypeScript
- Vite
- React Router
- styled-components

### Backend

- Node.js
- Express
- MySQL
- bcryptjs
- AWS S3 SDK

## 아키텍처

- `atto-front-ts`
  - 사용자/관리자 화면을 담당하는 프론트엔드 애플리케이션
- `atto-backend`
  - 인증, 상품, 장바구니, 주문, 배너, 관리자 API를 제공하는 백엔드 서버

프론트엔드는 `VITE_API_URL` 또는 same-origin 기준으로 API를 호출하도록 구성되어 있고, 백엔드는 단일 `server.js`에서 인증, 권한 처리, 검증, 상품/주문/회원 관련 API를 제공합니다.

## 구현 포인트

### 1. 사용자 영역과 관리자 영역을 분리한 커머스 구조

- 일반 사용자 라우트와 관리자 라우트를 구분하고, 로그인 여부와 관리자 권한에 따라 접근을 제어했습니다.
- 단순 쇼핑 화면 구현에 그치지 않고 운영 기능까지 포함해 서비스 전체 흐름을 볼 수 있도록 구성했습니다.

### 2. 실제 커머스 흐름을 고려한 주문 처리

- 장바구니, 결제 생성, 주문 생성 단계를 나누어 상태를 관리했습니다.
- 주문번호 생성, 주문 상태 변경, 결제 완료 처리 등 운영에서 필요한 후속 작업까지 반영했습니다.

### 3. 운영 편의성을 고려한 관리자 기능 구현

- 상품 등록/수정과 공개 여부 전환, 주문 상태 변경, 배너 관리, 회원 권한 변경 기능을 구현했습니다.
- 관리자가 직접 운영 상태를 확인하고 조작할 수 있도록 대시보드와 관리 화면을 구성했습니다.

### 4. 인증과 요청 보호

- 인증 토큰 발급 및 검증 로직을 직접 구현하고, 사용자/관리자 권한을 분리했습니다.
- 로그인/회원가입/Kakao 인증 요청에 대해 rate limiting을 적용하고, 보안 헤더와 쿠키 옵션을 설정했습니다.

## 실행 방법

### 1. Frontend

```bash
cd atto-front-ts
npm install
npm run dev
```

### 2. Backend

```bash
cd atto-backend
npm install
npm run dev
```

### 3. 환경 변수 예시

백엔드는 실행 전 아래 값들이 필요합니다.

```env
PORT=4000
HOST=0.0.0.0
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=atto_app
DB_PASSWORD=your_password
DB_NAME=atto
AUTH_TOKEN_SECRET=your_long_secret_key
AUTH_COOKIE_NAME=atto_auth
```

프론트엔드는 필요 시 API 주소를 아래와 같이 지정할 수 있습니다.

```env
VITE_API_URL=http://localhost:4000
```

## API 범위

백엔드에서 제공하는 주요 API 범위는 다음과 같습니다.

- 인증: 로그인, 로그아웃, 회원가입, Kakao 로그인/콜백
- 사용자: 프로필, 배송지, 찜, 장바구니
- 주문/결제: 주문 생성, 주문 조회, 주문 취소, 결제 생성
- 관리자: 대시보드, 사용자 관리, 상품 관리, 주문 관리, 결제 완료/환불 처리, 배너 관리

## 프로젝트에서 강조한 점

- 사용자 경험과 운영 기능을 따로 보지 않고 하나의 서비스 흐름으로 연결해 구현한 점
- 프론트엔드와 백엔드를 분리해 실제 배포 구조를 고려한 점
- 커머스 서비스에서 자주 다루는 인증, 장바구니, 주문, 관리자 운영 기능을 직접 설계한 점
- 화면 구현뿐 아니라 상태 변화와 운영 시나리오까지 포함해 서비스 관점으로 접근한 점

## 아쉬웠던 점과 확장 방향

- 백엔드 로직이 현재 단일 서버 파일 중심으로 구성되어 있어 도메인별 모듈 분리가 필요합니다.
- 테스트 코드와 CI 자동화가 추가되면 안정성과 협업 효율을 더 높일 수 있습니다.
- 결제 수단 다양화, 재고 정책 고도화, 이미지 업로드/미디어 관리 개선 등 실제 운영 수준의 확장이 가능합니다.

## 한 줄 회고

Atto는 단순한 쇼핑몰 UI 구현을 넘어서, 사용자 구매 흐름과 관리자 운영 흐름을 함께 설계하며 커머스 서비스의 전체 구조를 경험하는 데 초점을 둔 프로젝트입니다.
